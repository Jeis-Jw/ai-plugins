## 개요

PR #66 이후 남은 구조적 낭비를 제거한다. 실행 permit, command profile·impact rule, atomic physical claim, capability·telemetry·external-spend gate, generic adapter receipt, closeout reconciliation과 효율 보고를 구현한다.

## 근거

현재 review plan-next의 allowed_commands가 advisory이고 task-worker command profile 경로도 실제 schema·선택·claim을 강제하지 않아 동일 검증과 잘못된 명령이 실행될 수 있다. 세 독립 감사가 논리 gate가 아니라 physical execution 제어를 보강해야 한다고 합의했다.

## 범위와 완료 기준

완료 기준: native Studio와 task-worker 양쪽에서 동일 physical key 중복 실행이 차단되고 permit 밖 profile/raw command가 evidence와 closeout에 사용되지 않는다. integration HEAD full gate와 independent review는 유지한다. product-specific Vite/Render/Expo/Canvas 구현은 제외하고 declarative profile/adapter 계약만 제공한다. telemetry 미측정·외부 recurring spend·capability 실패는 명시 정책대로 fail-closed하며 closeout/효율 summary는 원장을 복제하지 않는 read-only projection이다. 전체 회귀·독립 리뷰·main 병합·버전업·Wiki/GitHub closeout까지 완료한다.
