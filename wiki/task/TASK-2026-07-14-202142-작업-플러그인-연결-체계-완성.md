## 개요

작업 플러그인 분리 이후 남은 provider binding, 중단 후 재개, GitHub Issue Tree 역방향 import, 외부 개발자 위임 모드와 설정 경계를 구현한다.

## 근거

작업 분해·병렬 실행·검증 강도는 유지하면서 세션 컨텍스트 의존, provider 상태 혼합, 동일 검증 반복과 closeout 복구 불가를 제거해야 한다.

## 범위와 완료 기준

범위: .task-worker.yml 신설과 .task-github.yml 분리, provider binding/context/run registry, Wiki TASK 직접 연결, Issue Tree import와 dispatch manual|worker, 통합/evidence·duplicate guard, 기존 facade migration, 전 플러그인 문서·테스트·버전 정합. 완료 기준: 다른 세션에서 TASK ID 또는 root Issue로 재개 가능, manual 위임과 worker 실행이 공존, 외부 closeout이 idempotent, 전체 회귀·wiki integrity 통과, main 통합·push 완료.
