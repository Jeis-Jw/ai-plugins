## 개요

task-worker에 provider-neutral WorkGraphSnapshot planner와 capability contract를 추가한다. task-github는 DefinitionArtifact/local lifecycle/ready 판단을 task-worker JSON CLI로 위임하고 GitHub projection, Issue/PR/CI/merge/closeout만 소유한다. 기존 task-github public skill과 legacy artifact 입력은 compatibility facade로 유지한다.

## 근거

사용자가 작업 관련 플러그인을 모두 완료하라고 지시했다. [[task-worker-plugin]]과 [[task-github-plugin]]의 목표 경계를 실제 runtime에 반영하되 Issue tree 분해, ready leaf 병렬성, worktree 격리, gear/review/merge gate, evidence reuse, integration 검증은 축소하지 않는다.

## 범위와 완료 기준

포함: task-worker WorkGraphSnapshot planner/capabilities/export, task-github task-worker discovery/preflight bridge, GitHub projection adapter, ready_leaves delegation, local facade skill 전환, compatibility CLI, 중복 core 제거, replay/contract 테스트, 양 플러그인 버전·marketplace·SSOT 갱신. 제외: Studio와 session-review 변경, 외부 GitHub mutation, 원격 push. 완료 조건: task-github가 provider-neutral scheduler/core를 자체 구현하지 않음, task-worker 미설치/계약 불일치 fail-closed, legacy CLI/artifact 입력 호환, ready/integration/gear 결과 회귀 없음, task-worker/task-github/전체 테스트와 plugin/skill validator 통과.
