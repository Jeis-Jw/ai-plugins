## 개요

사용자는 Studio를 기본 진입점으로 사용한다. Studio는 native harness로 리서치·기획·설계·구현·QA를 수행하며, 실행 파라미터나 .studio.yml에 명시된 경우에만 task-worker/task-github 및 session-review를 고수준 실행·독립 리뷰 edge에 선택적으로 사용한다.

## 근거

설정되지 않은 플러그인의 capability 탐색과 반복 실패, worker QA와 Studio QA의 이중 실행, 모든 리뷰에서 clean session을 강제하는 낭비를 제거한다. 병렬 ready-set, worktree 격리, 독립 검증, integration gate, 기존 planner/researcher/architect 역할은 유지한다.

## 범위와 완료 기준

완료 기준: (1) 도구 선택 우선순위가 run parameter > .studio.yml > native로 강제되고 미설정 플러그인은 조회하지 않는다. (2) worker/reviewer provider와 activation auto|always|never, fallback native|stop을 검증한다. (3) task-github 선택은 task-worker와 이중 lease하지 않는다. (4) reviewer는 위험 기반 edge에만 배치하고 finding follow-up은 동일 episode/evidence를 재사용한다. (5) Studio-owned `review_lease`는 task-worker와 task-github adapter가 모두 소비하며, task-github의 PR 생성·transport는 유지하되 reviewer dispatch만 억제한다. 두 플러그인의 standalone review 흐름은 유지한다. (6) .studio.yml은 공통 defaults/roles/agents/rituals와 providers.claude/providers.codex override를 지원한다. (7) 해석 우선순위와 provider-aware validation, host runtime 상속을 테스트한다. (8) native-only, configured tools, run override, unavailable fallback 조합의 conformance test를 통과한다. (9) Studio 0.5.0, task-worker 0.4.0, task-github 0.23.0의 adapter·lease 호환 계약과 Studio SSOT·README·producer 규약·manifest/marketplace 버전이 일치한다. (10) 결과가 main에 병합·push된다.

Review lease 정본은 exact `workflow-review-lease/v1` 10필드 `schema, lease_id, owner, provider, episode_id, edge_id, requirement, criteria_digest, evidence_refs, digest`다. owner는 `studio|task-worker`, provider는 `native|session-review`만 허용한다. Runtime profile은 verified `studio-runtime-capability/v1`와 일치할 때만 dispatch하고, advertised model/effort set이 있으면 fail-closed 검증하며 없으면 지원 상태를 `unknown`으로 남긴다.
