## 개요

플러그인마다 흩어진 scaffold/setup/진단 표면을 공통 init·doctor 계약으로 정리하되 native-first, provider-neutral 실행, GitHub adapter, 독립 review의 기존 책임을 보존한다.

## 근거

사용자가 승인한 권장안과 [[studio-plugin]], [[task-worker-plugin]], [[task-github-plugin]], [[session-review-plugin]]의 현재 경계를 근거로 한다. 초기화 편의가 외부 도구 자동 탐색이나 QA·병렬성 축소로 이어지지 않아야 한다.

## 범위와 완료 기준

포함:

- Studio의 분리된 workspace/config scaffold를 `studio:init` 하나로 통합하고 `--worker`, `--reviewer`, `--dry-run`, `--json`, `--force`를 제공한다. 미지정 시 native를 유지하고 외부 플러그인을 탐색하지 않는다.
- task-worker에 `task-worker:init`과 `task-worker:doctor`를 추가하고 `local`, `manual`, `quality`, `minimal` preset을 제공한다. 명령 profile과 impact rule은 추측하지 않고 명시적 TODO/fail-closed 상태로 만든다.
- task-github에 외부 mutation 없는 `task-github:init`을 추가하고 기존 `setup`은 Git/GitHub bootstrap으로 분리해 init을 재사용한다.
- session-review에는 별도 설정 SSOT를 만들지 않고 backend/wiki/git 준비 상태를 진단하는 `session-review:doctor`만 추가한다.
- 각 public skill, README/DESIGN, plugin SSOT, 테스트, 배포 manifest와 버전을 정합화한다.

완료 기준:

- 모든 init은 idempotent하고 기존 파일을 기본적으로 덮어쓰지 않으며, `--force`만 명시적 갱신을 허용한다.
- 모든 init은 `--dry-run`과 `--json`을 지원하고 생성 직후 자체 validator를 통과한다.
- init은 자기 플러그인 소유 경로만 변경하며 GitHub/외부 서비스 mutation을 수행하지 않는다.
- Studio는 run parameter > `.studio.yml` > native 우선순위와 미설정 provider 비탐색 원칙을 유지한다.
- task-worker는 분해, ready-set 병렬 실행, worktree 격리, 독립 검증, 통합 gate를 축소하지 않는다.
- task-github의 기존 setup/doctor 및 facade 호환이 유지되고 session-review는 새 지속 설정을 도입하지 않는다.
- 플러그인별 범위 검증 후 통합 HEAD 전체 회귀와 독립 리뷰를 1회 수행하고 finding 수정은 영향 범위만 증분 검증한다.
- 버전, marketplace manifest, Wiki SSOT를 맞춘 뒤 main에 병합·push하고 임시 branch/worktree를 정리한다.

영향 경로: `plugins/studio/**`, `plugins/task-worker/**`, `plugins/task-github/**`, `plugins/session-review/**`, `wiki/ssot/*-plugin.md`, `.agents/plugins/marketplace.json`, `.claude-plugin/marketplace.json`.

비목표: QA 축소, 병렬 분해 방법론 변경, 외부 plugin 자동 discovery, 제품별 test command 추측, wiki init 재설계, session-review 전용 설정 파일 신설.
