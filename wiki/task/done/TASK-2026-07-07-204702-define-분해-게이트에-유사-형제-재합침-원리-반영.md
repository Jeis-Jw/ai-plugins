## 개요

task-github define의 절단 판정을 [[DEC-2026-07-07-204311-분해-판정에-don-t-split-프로브와-재합침-우선-원리-도입]]대로 보강하는 단일 작업 단위. 텍스트(프로브·우선순위·challenge 기준), 메커니즘(dry-run 경고+테스트), 운영 규약(phase 체크리스트·재진입)을 phase 4개로 순차 수행한다. 절단 사유 4개 전부 불성립(P1/P2/P3 모두 define SKILL.md 접촉 = write-set 겹침, 저위험, 정보 경계 없음)이라 단일 이슈/단일 리프로 수행한다 — 이 구조 자체가 도입하는 원리의 자기 시연이다. 예상 gear: normal(start가 최종 판정).

## 근거

consumer dogfood #119(Lightning Santa UI, plugin 0.18.1): 절단 원리 텍스트가 존재하는 버전에서 #121/#122/#123이 same-theme 형제로 과분해돼 #122 sibling merge conflict, 동일 검증 3회 반복, 리프 3개분 세리머니가 발생했다. 룰 부재가 아니라 사유①(병렬 이득)의 '독립 조각' 판정 실패 — 상세 사례와 결정 근거는 [[DEC-2026-07-07-204311-분해-판정에-don-t-split-프로브와-재합침-우선-원리-도입]] 배경 절. 반려 대안 2건: [[REJ-2026-07-07-204429-orchestrate-실행-단계에서-여러-이슈를-한-워커에-묶어-처리]], [[REJ-2026-07-07-204429-이슈-분할-판정을-3-of-5-산술-기준으로-대체]].

## 범위와 완료 기준

P1 — define 절단 판정 보강(텍스트): plugins/task-github/skills/define/SKILL.md 절단 원리에 (a) 헤드라인 질문 '이 조각을 다른 워커가 독립 점유해 끝까지 끌고 갈 수 있는가' (b) don't-split 프로브 3개 — 형제 검증 명령 사실상 동일 / 같은 shared component·기반 수정('N surfaces × same system' 패턴은 공유 기반이 곧 write-set, 표면 디렉토리 분리에 속지 말 것) / 앞 조각 context를 뒤 조각이 계속 필요 — 하나라도 해당하면 사유① 불성립으로 묶기 (c) 겹침 처리 우선순위: same-theme write-set 겹침 → 재합침(한 리프+phase) 우선, blocked_by 직렬화는 다른 deliverable일 때 차선(직렬 리프 N개 = phase N개 + 세리머니 N배 명시) (d) §challenge review (c)의 근거 기준에 don't-split 프로브 추가(가짜 독립 감사) (e) rules/quality-gates.md G4 '겹침+dependency 미선언' 항목에 재합침 우선 검토 문구.

P2 — siblings_maybe_phases dry-run 경고(메커니즘): plugins/task-github/skills/define/scripts/create_issue_tree.py에 _detect_flat_understructuring(98행 부근) 미러로 _detect_siblings_maybe_phases 추가. 신호 2+개로 발동: (a) 같은 parent의 리프 3+개가 동일한 단일 선행 리프에만 blocked_by(그 외 상호 dep 없음) (b) 형제 affects_paths 공통 경로 교집합 또는 단일 클러스터 수렴 (c) 제목 same-theme 키워드가 형제 3+개에 반복 (d) 리프 본문 '검증:' anchor 명령이 형제 간 동일. 산출은 비차단 warnings[] 항목 {code: siblings_maybe_phases, message, suggested_merge: [리프 키들]} — STOP 아님, 기존 경고 규약 동일. SKILL.md 경고 목록에 항목 추가 + 경고 시 'N개 리프를 1리프+phase로 합치는 안'을 사령관 확인안에 병기하도록 지시. tests/test_execution_contract.py에 테스트: #119 형태 입력(선행 1 + same-theme 형제 3 + 공통 기반 경로) → 발동, 독립 형제(경로·검증 분리) → 미발동.

P3 — phase 운영 규약(텍스트): define SKILL.md granularity 정지 규칙 확장 — 리프 본문 phase 체크리스트 규약(phase별 커밋, phase 완료마다 체크박스+체크포인트 코멘트, 마지막 phase=full-verify 1회). skills/run/SKILL.md에 phase 재진입 규약 — 긴 리프는 phase별 순차 워커 세션으로 이어받기 가능(같은 이슈/브랜치/worktree 유지, run-notes로 학습 승계), 세리머니(이슈·PR·closeout·머지엣지)는 1회 유지, compaction 방어.

P4 — 정합·검증: 전체 테스트 + define ↔ quality-gates ↔ run 상호참조 정합 확인.

완료 기준: (1) don't-split 프로브·재합침 우선순위·헤드라인 질문이 define SKILL.md에 반영되고 challenge review 기준에 포함된다 (2) siblings_maybe_phases 경고가 #119 재구성 입력에서 발동하고 독립 형제에서 미발동하는 테스트가 green (3) phase 운영 규약이 define/run SKILL.md에 존재 (4) 전체 pytest green.

검증: python3 -m pytest plugins/task-github/tests/ -q
