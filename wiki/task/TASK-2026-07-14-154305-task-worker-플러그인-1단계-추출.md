## 개요

task-github 0.20.0에 함께 들어 있는 provider-neutral DefinitionArtifact와 local execution lifecycle을 독립 task-worker 플러그인으로 추출한다. 분해, ready-set 병렬성, worktree 격리, 검증 evidence라는 기존 엔지니어링 방법론은 축소하지 않는다.

## 근거

사용자 지시와 [[task-worker-plugin]], [[task-github-plugin]] 설계 정본을 근거로 한다. 플러그인 분리 자체가 새 agent/session hop이나 검증 약화를 만들지 않아야 한다.

## 범위와 완료 기준

포함: task-worker plugin scaffold, provider-neutral artifact/run core, legacy schema 입력 호환, local define/start/run/verify/done/status/orchestrate skill, ready action set planner, 테스트, marketplace 배포 메타데이터, SSOT 현재 상태 갱신. 제외: Studio 변경, task-github GitHub adapter/facade 위임 전환, GitHub projection 이동, remote PR/merge 변경. 완료 조건: task-worker 단독 테스트 통과, plugin/skill/marketplace 검증 통과, task-github 기존 테스트 회귀 없음, GitHub 호출 없는 provider-neutral 경계 확인.
