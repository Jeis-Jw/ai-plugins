## 개요

`wiki_cli`의 다중값 관계 플래그를 `action="append"` + 콤마 split·flatten·strip·순서보존 dedup으로 통일한다. 현재 반복 지정 시 마지막 값만 남고 앞 값이 조용히 드롭되는 동작을 제거 — 반복·콤마·혼합·중복 4형을 모두 정상 처리한다.

## 근거

[[OBS-2026-06-12-190117-wiki-cli-다중값-플래그-반복-시-조용히-마지막만-남김]] 참조. 관계 링크의 조용한 누락은 결정 그래프 무결성을 손상하고(예외 없는 결정성 원칙 위반), immutable record라 `relate` 사후 보강도 막혀 복구가 비싸다. 채택안은 **(a) 누적** — 사용자가 두 값을 주면 의도가 명확(둘 다)하므로 받아주는 게 (b) 에러 거부보다 적합. 영향 surface는 [[wiki-data-model]](관계 무결성).

## 범위와 완료 기준

### 범위

- **공유 정규화 헬퍼**: `action="append"`로 수집 → 각 값 콤마 split → flatten → strip → 순서보존 dedup.
- **전 list형 관계 인자에 일괄 적용**: `--intents` / `--decisions` / `--ssot` / `--runbook` / `--rejected` / `--tasks` / `--add-intents` / `--add-decisions` / `--add-ssot` / `--add-tasks`.

### 완료 기준

- 4형 모두 정확한 set 산출:
  - 반복형 `--intents a --intents b` → `[a, b]`
  - 콤마형 `--intents a,b` → `[a, b]` (기존 동작 비파괴)
  - 혼합형 `--intents a,b --intents c` → `[a, b, c]`
  - 중복 `--intents a --intents a` → `[a]` (멱등)
- **조용한 드롭 0** — 반복 시 누락 없음.
- 위 4형 회귀 테스트 존재.

### 범위 밖

- 비관계 스칼라 인자(`--title`/`--summary`/`--slug` 등)는 대상 아님.
