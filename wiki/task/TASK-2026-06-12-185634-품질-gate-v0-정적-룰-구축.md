## 개요

task-github / wiki-markdown에 결정·분해 품질 gate의 **v0(정적 룰)**를 구축한다. 비동기 실행 전에 나쁜 define/decision을 걸러 사람 확인 비용을 낮추는 안전장치. 모든 체크는 결정적·model-agnostic이며, 위반 시 차단(G1) 또는 FLAG-to-human(G2/G3)으로 동작한다. LLM-judge(v1, 의미판정)는 이 작업 범위 밖이다.

## 근거

[[DEC-2026-06-12-185228-결정-분해-품질-gate를-플러그인에-추가-정적-룰-v0-먼저]] 참조. 빠른 define 후 비동기 실행 모델에서는 사람이 실행 루프에 없어 나쁜 define이 PR에서야 드러난다 — 비동기 핸드오프가 define 품질 기준을 오히려 높인다. 현재 플러그인은 구조·스키마·무결성은 갖췄으나 결정·분해의 의미적 품질 gate가 없다. 정적 룰을 먼저 하는 이유(결정적·천장 없음·저렴)와 LLM-judge-first 반려 근거는 결정·반려 노드에 있다.

## 범위와 완료 기준

### 범위 (v0 정적 룰)

- **G1 — refresh hard gate**: `refresh --strict` / `changed-path-stale`를 verify·done·merge에서 보고(report)→**차단(hard gate)**으로 전환. (배선 이미 존재, 제일 작음)
- **G2 v0 — decision 정적 품질 체크**: 근거 섹션 non-trivial 길이 / 반려대안 ≥1 / intent·관계 링크 resolve / 트레이드오프 non-empty / 재평가조건 존재.
- **G3 v0 — define/분해 정적 품질 체크**: 각 유닛 완료기준 non-empty / verification 명시 / touched·affects-path 명시 / path 겹침 시 dependency 강제 / parent–leaf altitude mismatch 탐지.
- **G4 — escalation 기준**: major·uncertain 작업에 FLAG-to-human 기준 정의(차단 아님, 정책성).

### 완료 기준

- G1 hard gate가 staleness/무결성 위반 시 verify·done·merge를 **실제로 차단**한다.
- G2/G3 정적 체크가 위반 시 **FLAG-to-human**을 출력하고(차단 기본값 아님), 통과 시 무음.
- 각 체크에 **회귀 테스트**(통과/실패 fixture)가 존재한다.
- 모든 룰이 결정적·model-agnostic — 특정 에이전트/모델 이름에 비결합([[INT-2026-05-29-104711-plugin-agent-neutrality]] 준수).
- G4 escalation 기준이 문서화되어 major·uncertain 판정에 적용 가능.

### 범위 밖

- LLM-judge(v1): 의미판정 evaluator, evidence-first, calibration corpus. 정적 룰 v0가 못 거르는 의미적 얕음이 실측으로 드러날 때 별도 작업으로.
