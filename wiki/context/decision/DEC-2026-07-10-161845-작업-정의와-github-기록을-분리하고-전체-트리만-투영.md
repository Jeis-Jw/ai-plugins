## 결정

define의 정본 산출물을 안정적인 node id와 digest를 가진 DefinitionArtifact로 둔다. 실행은 이를 로컬에서 직접 소비할 수 있다. GitHub 기록을 선택한 경우 root와 모든 descendant 및 dependency를 전부 등록한 뒤에만 실행하며 partial/sparse projection은 허용하지 않는다.

## 취지

작업 분해와 실행 품질을 GitHub 사용 여부와 분리하면서도, 기록을 선택한 업무는 중간 누락 없는 완전한 IssueTree로 보존한다.

## 배경

기존 define은 작업정의와 GitHub Issue 생성을 한 절차에 묶어 작은 작업에도 원격 기록 비용을 부과했다. 반대로 선택적 일부 투영은 기록의 의미를 흐릴 수 있어 all-or-none 경계가 필요하다.

## 고려한 대안

기존 Issue-first 유지, 특정 subtree만 선택 투영, execution unit마다 개별 Issue를 만드는 방식을 검토했다.

## 트레이드오프

로컬 실행은 원격 복구성이 낮으므로 ignored local ledger와 digest 검증이 필요하다. GitHub 전체 투영은 Issue 수가 늘 수 있으므로 semantic node의 절단 기준을 보수적으로 유지한다.

## 재평가 조건

DefinitionArtifact와 GitHub IssueTree 사이의 복구가 반복적으로 실패하거나, 로컬 정의가 GitHub 모드와 다른 실행 의미를 갖게 되면 경계를 재검토한다.
