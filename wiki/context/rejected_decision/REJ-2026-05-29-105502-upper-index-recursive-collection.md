## 대안

상위 인덱스가 하위 폴더의 활성 문서 summary까지 모두 재귀 수집하는 방식이다. 예를 들어 `ssot/ssot.md`가 `ssot/auth/auth-session.md`와 `ssot/payment/payment-flow.md`까지 모두 보여준다.

## 반려 사유

Nested 폴더를 도입한 이유는 영역별로 인덱스와 문맥을 분할하기 위해서다. 상위 인덱스가 하위 문서를 모두 재귀 수집하면 상위 인덱스가 다시 비대해지고, 같은 문서가 여러 인덱스에 중복 노출된다.

최종 규칙은 "재귀 = 폴더 발견, 비재귀 = 노트 수집"이다. 각 폴더 인덱스는 자기 직속 문서만 모은다.

## 이 대안의 취지

상위 인덱스 하나로 전체 하위 문서를 조망하려는 목적이었다. 사용자가 하위 폴더를 일일이 내려가지 않아도 전체 목록을 볼 수 있다는 장점이 있다.

## 재고 조건

전체 조망이 필요하면 상위 인덱스에 하위 인덱스 링크나 별도 map 문서를 둔다. 하위 문서 summary 재귀 수집은 토큰 효율과 중복 문제 때문에 기본 정책으로 재도입하지 않는다.
